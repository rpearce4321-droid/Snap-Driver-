# Admin Use-Case SOP

Last updated: 2026-02-08

## Purpose
Operate the Admin portal to manage users, approvals, badges, work units, and system health for Snap Driver.

## Core access
1. Open the Admin portal and confirm you are in the `ADMIN` context.
2. Verify the header panel name matches the work you are about to do.

## User management
1. Use email as the username for all accounts.
2. Create invite links when onboarding new users.
3. Set or reset passwords as needed. Passwords are never displayed.
4. Update user status only with a status note when deactivating.

## Seeker approvals
1. Review pending seekers for completeness and policy compliance.
2. Approve, reject, or delete based on the submission status.
3. If a seeker is deleted or rejected, ensure linking is broken as required.

## Retainer approvals
1. Review pending retainers for completeness and policy compliance.
2. Approve, reject, or delete based on the submission status.
3. If a retainer is deleted or rejected, ensure linking is broken as required.

## Content and routes
1. Review routes for accuracy and audience settings.
2. Review posts and broadcasts for policy compliance.

## Badge center and scoring
1. Review badge rules for role-specific requirements.
2. Adjust scoring settings only when approved by product decisions.
3. Use audit views to spot inconsistencies.

## Work units and disputes
1. Review disputed work unit periods.
2. Confirm, neutralize, or resolve disputes based on evidence.
3. Ensure dispute notes are recorded.

## Messaging traffic
1. Review message traffic panels for safety and compliance.
2. Escalate issues per policy.

## Server and seed (staging only)
1. Use seed actions only in staging or local testing.
2. Avoid wiping data unless explicitly approved.
